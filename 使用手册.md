# 王明课题组网站中文使用手册

本手册面向课题组日常维护人员，按“能直接上手”编写。

## 1. 网站现有结构

当前网站是多页面结构：

- `index.html`：首页
- `members.html`：成员列表与简介页
- `publications.html`：论文列表页
- `paper.html`：单篇论文详情页（通过 `?id=...` 打开）
- `news.html`：新闻列表页
- `news-item.html`：单条新闻详情页（通过 `?id=...` 打开）

脚本文件：

- `site-data.js`：核心数据（双语文案、成员、论文、新闻）
- `script.js`：首页逻辑
- `members.js`：成员页逻辑
- `publications.js`：论文列表逻辑
- `paper.js`：论文详情逻辑
- `news.js`：新闻列表逻辑
- `news-item.js`：新闻详情逻辑

样式文件：

- `styles.css`

照片目录：

- `assets/members/`

## 2. 最常用的三类修改

### 2.1 修改成员信息

编辑文件：`site-data.js`  
数据位置：`const MEMBER_DATA = { ... }`

在读博士生示例：

```js
{
  name: "王绍志",
  enName: "Shaozhi Wang",
  email: "wangshaozhi@wangming-lab.com",
  focusZh: "金属有机笼精准构筑",
  focusEn: "Precise construction of metal-organic cages",
  bioZh: "关注配体工程与结构可控组装。",
  bioEn: "Focuses on ligand engineering and controllable assembly."
}
```

毕业成员示例：

```js
{
  name: "马建军",
  enName: "Jianjun Ma",
  destinationZh: "已毕业",
  destinationEn: "Graduated"
}
```

### 2.2 修改论文信息

编辑文件：`site-data.js`  
数据位置：`const PAPER_DATA = [ ... ]`

单篇论文字段：

```js
{
  id: "paper-2024-01",
  date: "2024",
  zh: {
    title: "中文/英文题目",
    journal: "期刊信息",
    authors: "作者",
    abstract: "中文摘要",
    keywords: "中文关键词"
  },
  en: {
    title: "Title",
    journal: "Journal",
    authors: "Authors",
    abstract: "English abstract",
    keywords: "English keywords"
  },
  source: "https://doi.org/xxxx"
}
```

注意：

- `id` 必须唯一，且不要改成重复值。
- `source` 建议填 DOI 或 PDF 链接，不能留空。

### 2.3 修改新闻信息

编辑文件：`site-data.js`  
数据位置：`const SITE_NEWS = [ ... ]`

单条新闻字段：

```js
{
  id: "news-20260320-1",
  date: "2026-03-20",
  zh: {
    title: "中文标题",
    desc: "中文摘要",
    detail: ["正文第一段", "正文第二段"]
  },
  en: {
    title: "English title",
    desc: "English short description",
    detail: ["Paragraph 1", "Paragraph 2"]
  }
}
```

## 3. 双语维护规则

所有新内容都要同时写：

- `zh`：中文版本
- `en`：英文版本

如果只写中文，英文页面会出现缺失信息。

## 4. 成员照片如何放

把照片放到：

- `assets/members/`

建议命名：

- 与成员 `name` 一致，如 `王绍志.jpg`

如果没有对应照片，页面会自动显示姓名首字占位图，不会报错。

## 5. GitHub 上传与发布

## 5.1 上传步骤

1. 打开仓库主页  
2. 进入 `Add file` -> `Upload files`  
3. 上传本地 `wangming-lab-site` 中所有更新后的文件  
4. `Commit changes`

## 5.2 发布设置

1. `Settings` -> `Pages`  
2. `Deploy from a branch`  
3. 分支选 `main`，目录选 `/ (root)`  
4. 保存，等待 1-5 分钟

## 5.3 访问链接

通常格式：

- `https://你的用户名.github.io/仓库名/`

## 6. 常见问题排查

### 6.1 页面显示旧内容

- 按 `Ctrl + F5` 强制刷新
- 确认 GitHub 已成功提交新文件

### 6.2 404

检查仓库根目录是否有这些文件：

- `index.html`
- `members.html`
- `publications.html`
- `news.html`

### 6.3 点击没反应

检查 `site-data.js`：

- 论文 `id`、新闻 `id` 是否唯一
- 链接是否写错

### 6.4 中文乱码

确保文件编码为 UTF-8（无 BOM 或 UTF-8 均可）。

## 7. 推荐维护流程（每月）

1. 更新本月新闻 1-3 条（`SITE_NEWS`）  
2. 新增论文（`PAPER_DATA`）  
3. 更新成员去向/新增成员（`MEMBER_DATA`）  
4. 上传照片  
5. 提交并发布

## 8. 你们后续只要改哪里

绝大多数内容只改一个文件：

- `site-data.js`

如果只改数据，通常不用改 `html/css/js` 逻辑文件。
