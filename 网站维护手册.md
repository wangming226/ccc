# 王明课题组网站维护手册

这份手册是给课题组后续自己维护网站用的，重点解决 4 件事：

1. 怎么换 `Logo`
2. 怎么换 `老师照片`
3. 怎么加 `成员照片`
4. 怎么改 `课题组介绍 / 新闻 / 论文 / 仪器设备`

## 二、Logo 怎么换

### 1. Logo 文件放哪里

把课题组 Logo 放到：

`assets/logo.png`

建议：

- 文件名固定用 `logo.png`
- 背景尽量透明
- 建议尺寸：`600 x 600` 或更大
- 图片不要太大，建议控制在 `1MB` 以内

### 2. 如果你们的 Logo 不是 png

也可以用 jpg 或 svg，但需要同时改 `site-data.js` 里的路径：

找到：

```js
const PI_DATA = {
  photo: "./assets/members/王明.jpg",
  logo: "./assets/logo.png"
};
```

如果你换成 `logo.jpg`，就改成：

```js
logo: "./assets/logo.jpg"
```

## 三、老师照片怎么换

### 1. 老师照片放哪里

把王明老师照片放到：

`assets/members/王明.jpg`

建议：

- 文件名用 `王明.jpg`
- 建议尺寸：`800 x 1000` 左右
- 人像尽量竖图
- 背景尽量简洁

### 2. 如果文件名不是这个

同样改 `site-data.js`：

```js
const PI_DATA = {
  photo: "./assets/members/王明.jpg",
  logo: "./assets/logo.png"
};
```

例如你想换成：

`assets/members/prof-wang.jpg`

那就改成：

```js
photo: "./assets/members/prof-wang.jpg"
```

## 四、成员照片怎么放

### 1. 所有成员照片都放这里

`assets/members/`

### 2. 推荐命名方式

建议直接用中文姓名命名，最省事：

- `王绍志.jpg`
- `谌宏群.jpg`
- `张馨睿.jpg`
- `郭梓腾.jpg`

### 3. 照片显示规则

成员页当前是按数据中的姓名或路径去找图片。后续如果你希望完全自动按“姓名匹配图片”，我也可以再帮你补一版。

目前稳妥做法是：

在 `site-data.js` 的成员数据里，给每位成员加一个 `photo` 字段，例如：

```js
{
  name: "郭梓腾",
  enName: "Ziteng Guo",
  email: "guozt22@mails.jlu.edu.cn",
  photo: "./assets/members/郭梓腾.jpg",
  focusZh: "功能有机-金属组装体",
  focusEn: "Functional organic-metal assemblies",
  bioZh: "聚焦发光与电子功能耦合。",
  bioEn: "Focuses on coupled luminescent and electronic functions."
}
```

如果不加 `photo`，网站会显示默认占位块，不会报错。

## 五、首页新闻轮播怎么改

首页新闻轮播的数据在：

`site-data.js`

主要看这两部分：

### 1. 新闻正文数据

```js
const SITE_NEWS = [
  {
    id: "news-20260320-1",
    date: "2026-03-20",
    tags: ["网站更新", "信息架构", "多页面"],
    zh: {
      title: "网站升级为多页面结构",
      desc: "成员、论文、新闻均可独立跳转访问。",
      detail: [
        "第一段",
        "第二段"
      ]
    },
    en: {
      title: "Website upgraded to multi-page structure",
      desc: "Members, publications, and news now have standalone pages.",
      detail: [
        "First paragraph",
        "Second paragraph"
      ]
    }
  }
];
```

### 2. 首页轮播选哪些新闻

```js
const HERO_SLIDES = [
  {
    newsId: "news-20260320-1",
    accent: "信息架构"
  }
];
```

意思是：

- `SITE_NEWS` 负责存全部新闻
- `HERO_SLIDES` 负责指定首页轮播展示哪些新闻

如果你新加了一条新闻，只要：

1. 先在 `SITE_NEWS` 新增该新闻
2. 再把它的 `id` 写到 `HERO_SLIDES`

## 八、论文怎么改

论文数据在：

`site-data.js`

格式：

```js
const PAPER_DATA = [
  {
    id: "paper-2024-01",
    date: "2024",
    zh: {
      title: "中文或英文题目",
      journal: "期刊名",
      authors: "作者",
      abstract: "摘要",
      keywords: "关键词"
    },
    en: {
      title: "English title",
      journal: "Journal",
      authors: "Authors",
      abstract: "Abstract",
      keywords: "Keywords"
    },
    source: "#",
    tags: ["超分子化学", "金属有机笼"]
  }
];
```

如果有 DOI 或原文链接，把 `source` 从 `#` 改成真实链接：

```js
source: "https://doi.org/xxxxx"
```

## 九、双语怎么改

网站的中英双语文案主要在：

`site-data.js`

结构是：

```js
const SITE_I18N = {
  zh: {
    heroTitle: "王明课题组"
  },
  en: {
    heroTitle: "Wang Ming Laboratory"
  }
};
```

规则很简单：

- 中文写到 `zh`
- 英文写到 `en`
- 两边字段名必须一致

如果你只改中文、不改英文，那么切到英文页面时就可能显示不完整。

## 十、修改后怎么更新到 GitHub

### 1. 本地先改文件

改完：

- `site-data.js`
- `assets/` 里的图片

### 2. 上传到 GitHub 仓库根目录

把改好的文件重新上传覆盖原文件。

### 3. 等 GitHub Pages 自动部署

一般 1 到 3 分钟。

### 4. 强制刷新网页

按：

`Ctrl + F5`

## 十一、常见问题

### 1. Logo 不显示

检查：

- 文件是否真的叫 `logo.png`
- 是否放在 `assets/` 下
- `site-data.js` 的路径是否写对

### 2. 老师照片不显示

检查：

- 文件是否放在 `assets/members/`
- 文件名是否和路径一致
- 中文文件名有没有写错

### 3. 上传后网页没变化

检查：

- GitHub 是否完成 Pages 部署
- 是否强刷 `Ctrl + F5`

### 4. 英文页面内容没变

检查：

- 是否同时修改了 `zh` 和 `en`
